# MVP Boundaries — SloDocs

## Цель MVP
Запустить минимально рабочую версию сервиса с одной платной услугой,
которая позволяет пользователю автоматически получить доступ
к информационным материалам без личного общения.

## Что ВХОДИТ в MVP
- минимальная административная часть для добавления и редактирования услуг
- одна услуга: «Оформление ребёнка в школу в Словении»
- главная страница со списком услуг
- страница услуги с кратким описанием что входит в материалы и кнопкой для оплаты/доступа
- онлайн-оплата доступа
- запрос email для предоставления доступа и уведомлений
- автоматический доступ к материалам оплаченной услуги
- ограниченный срок доступа к материалам
- уведомления, связанные с оказанием услуги (оплата, окончание доступа)
- базовое сообщение об ошибке оплаты

## Нефункциональные требования (качество MVP)

- базовая безопасность доступа к материалам
- ограничение доступа по сроку и оплате
- отсутствие публичных прямых ссылок на материалы
- использование HTTPS
- минимальная защита административной части

- соблюдение базовых требований доступности интерфейса
- семантическая HTML-разметка
- читаемость и доступность контента

- базовая SEO-оптимизация
- корректная структура страниц
- использование schema.org
- человекочитаемые URL

## Наблюдаемость и контроль
- логирование ошибок оплаты
- логирование ошибок доступа
- фиксация ключевых событий (оплата, выдача доступа, окончание срока)

## Обработка ошибок
- понятные сообщения при ошибке оплаты
- понятное сообщение при истечении срока доступа
- отсутствие «пустых экранов»

## Контентная модель
- контент услуги хранится структурировано
- возможность редактировать текст без изменения кода

## Надёжность
- резервное копирование базы данных
- возможность восстановления данных

## Тон и формулировки
- нейтральный информационный тон
- отсутствие обещаний результата
- отсутствие формулировок «гарантируем», «обязаны»

## Контроль масштаба
- любые новые функции добавляются только после проверки критериев успеха MVP

## Модель данных (минимальная)
- услуга
- покупка
- email пользователя
- срок доступа
- статус оплаты

## Платежи
- обработка повторных webhook-событий без дублирования доступа
- одна покупка = один активный доступ

## Архитектурные ограничения
- бизнес-логика вне шаблонов
- отсутствие логики доступа в представлениях
- контроллеры тонкие

## Конфигурация
- все ключевые параметры через env
- срок доступа
- email-отправитель
- платежный провайдер

## Поведение при сбоях
- при ошибке оплаты доступ не выдаётся
- при ошибке email доступ всё равно сохраняется
- при ошибке доступа пользователь видит понятное сообщение

## Зависимости
- минимальное количество сторонних пакетов
- отсутствие необязательных JS-библиотек

## Тестируемость
- возможность пройти основной User Flow в тестовом режиме
- тестовый платёж

## Осознанно не используется
- SPA
- сложные JS-фреймворки
- realtime-обновления

## SEO: структура сайта
- каждая услуга имеет отдельный URL
- плоская структура без лишних уровней
- логическая иерархия страниц

## SEO: доступность контента
- описания услуг и вводные тексты доступны без оплаты
- платный контент закрыт от индексации

## SEO: метаданные
- уникальный title для каждой страницы услуги
- уникальный description
- отсутствие дублирующихся метатегов

## SEO: контент
- тексты пишутся для людей, не для ключей
- один основной поисковый интент на страницу
- отсутствие keyword stuffing

## SEO: структурированные данные
- использование schema.org
- типы: WebSite, Service, FAQ (при наличии)

## SEO: индексация
- корректный robots.txt
- sitemap.xml
- запрет индексации служебных и платных страниц

## SEO: производительность
- отсутствие тяжёлых JS-бандлов
- минимальный клиентский JS
- приоритет серверного рендеринга

## SEO: региональность
- указание языка страниц
- ориентация на регион Словения

## SEO: осознанно не используется
- автоматическая генерация текстов ради SEO
- дорвеи
- скрытые тексты
- покупные ссылки

## Юридические страницы
- страница «Условия использования»
- страница «Политика конфиденциальности»
- указание, что сервис носит информационный характер

## Контент и версии
- фиксированная версия контента на момент покупки
- обновления контента не аннулируют уже выданный доступ

## Ограничение ответственности
- пользователь самостоятельно принимает решения на основе материалов
- сервис не несёт ответственности за действия государственных органов

## Что НЕ ВХОДИТ в MVP
- личный кабинет пользователя
- регистрация и пароли
- поиск по сайту
- рейтинги, отзывы и комментарии
- скидки, промокоды и бонусы
- маркетинговые email-рассылки
- сопровождение и консультации
- юридические услуги
- гарантии результата
- мобильное приложение
- многоязычность интерфейса

## Осознанно отложено
- добавление новых услуг
- поиск по сайту
- многоязычность интерфейса (русский, английский)
- личный кабинет пользователя
- подписки и пакеты услуг
- бонусные программы
- расширенная аналитика
