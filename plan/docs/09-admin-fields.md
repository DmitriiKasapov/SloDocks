# Admin Fields — SloDocs (MVP)

## Назначение документа
Зафиксировать поля и блоки, доступные администратору
для управления услугами и контентом в MVP.

Документ:
- не описывает UX и интерфейс
- не определяет внешний вид форм
- фиксирует структуру данных, доступных в админке

Документ зафиксирован для MVP.

---

## Управление услугой (Service)

Администратор может редактировать следующие поля услуги.

### Основные данные
- title — заголовок услуги
- slug — используется для URL
- description_public — публичное описание (до оплаты)
- price — цена доступа
- access_duration_days — срок доступа
- is_active — включение / отключение услуги

Примечания:
- `slug` уникален
- отключение услуги не аннулирует уже выданные доступы

---

## Контент услуги (платный контент)

Контент услуги состоит из структурированных блоков.

### 1. Заголовок услуги
- используется:
  - на странице услуги
  - в SEO
  - в email-уведомлениях

---

### 2. Публичное описание
- краткое описание услуги
- отображается до оплаты
- используется для SEO

---

### 3. Инструкция
- основной пошаговый текст
- структурированный (разделы, подпункты)
- доступен только после оплаты

---

### 4. Документы (PDF)

Администратор может прикреплять:
- PDF — чистые бланки
- PDF — заполненные образцы

Примечания:
- файлы не имеют публичных URL
- файлы доступны только после проверки доступа
- количество файлов не ограничено

---

### 5. Список ссылок
- ссылки на официальные источники
- сайты учреждений
- онлайн-формы

Примечания:
- ссылки могут вести на внешние ресурсы
- список редактируется администратором

---

### 6. Практические советы
- текстовый блок
- типичные ошибки
- нюансы из практики
- рекомендации

---

### 7. Медиа (опционально, MVP+)
- изображения (сканы, схемы, примеры)
- карта (адрес учреждения, Google Maps / OpenStreetMap)

Примечания:
- медиа не являются обязательными
- отсутствие медиа не блокирует публикацию услуги
- реализация медиа допускается после MVP

---

## Просмотр пользователей и покупок (только чтение)

Администратор может просматривать агрегированные данные:

### Пользователи (по email)
- email
- общая сумма покупок
- количество покупок
- список купленных услуг
- дата последней покупки

Примечания:
- данные предназначены для статистики
- редактирование запрещено

---

## Поля, недоступные для редактирования

Администратор НЕ может редактировать:
- id
- created_at
- updated_at
- данные покупок
- статусы платежей
- данные доступов
- агрегированную пользовательскую статистику

---

## Поведение изменений

- изменение контента не влияет на уже выданные доступы
- изменение цены не влияет на ранее оплаченные покупки
- отключение услуги не аннулирует активные доступы

---

## Что осознанно отсутствует

- визуальный редактор
- версии контента
- массовые операции
- импорт / экспорт
- управление пользователями
- управление платежами
